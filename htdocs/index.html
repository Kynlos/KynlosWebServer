Kynlos Python Web Server

A full-featured web server with PHP support

Features

check_circle

Static File Serving

Serves static files from the current directory, allowing you to host HTML, CSS, JavaScript, images, and other static content.

check_circle

PHP Support

Executes PHP scripts using the php command, enabling dynamic content generation and server-side scripting.

check_circle

Configurable

Highly configurable via a config.json file, allowing you to set the port, entry point, and other settings.

check_circle

Logging

Detailed logging of requests and errors to a log file, helping you monitor and debug your server.

check_circle

Directory Listing

Automatically generates a directory listing for directories, making it easy to browse files and folders.

check_circle

File Upload

Supports file uploads via POST requests to /upload, with uploaded files saved in the specified directory.

check_circle

Rate Limiting

Implements rate limiting to prevent abuse, configurable to allow a specific number of requests per minute per client. You can configure rate limiting, including burst limits, in the config.json file.

check_circle

HTTPS Support

Supports HTTPS with automatic self-signed certificate generation, enhancing security for your web server. Configure HTTPS settings, including certificate details and renewal options in config.json.

check_circle

CORS Support

Enables Cross-Origin Resource Sharing (CORS), allowing your server to handle requests from different origins.

check_circle

IP Whitelisting/Blacklisting

Allows or blocks access based on client IP addresses, providing an additional layer of security. Manage whitelisted and blacklisted IP addresses via the config.json file.

check_circle

Custom Error Pages

Serves custom HTML pages for different HTTP error codes, improving the user experience during errors. Configure your custom error pages in the config.json file.

check_circle

Static File Caching

Implements caching for static files to improve performance and reduce server load. You can configure caching behavior, including cache size and TTL (time-to-live), in config.json.

check_circle

Gzip/Brotli Compression

The server can automatically compress files using Gzip or Brotli to reduce file sizes and improve transfer speeds, leading to faster page loads.

check_circle

Basic Authentication

Secure your server by enabling basic authentication and defining authorized usernames and passwords in the config.json file.

check_circle

Virtual Hosts

Host multiple websites or domains on a single server using virtual hosts. Define your virtual host mappings in config.json to route requests accordingly.

check_circle

URL Redirects

Easily redirect users from old or outdated URLs to new ones, or to external websites, using the config.json file.

check_circle

Graceful Shutdown

Supports graceful shutdown on Ctrl-C, ensuring that all processes are properly terminated.

check_circle

Browser Auto-Open

Automatically opens the entry point in a web browser on server start, providing a convenient user experience.

check_circle

Email Verification

Provides email verification for new user registrations, adding an extra layer of security and preventing spam accounts.

check_circle

SQLite Database Integration

Uses an SQLite database to manage user accounts, enabling features like user registration and login.

Getting Started

  1. Place your web files in the same directory as the server script.
  2. Ensure you have PHP installed and accessible from the command line.
  3. Run the server using the command: python main.py
  4. Open your browser and navigate to http://localhost:8080 (or the port you configured).

File Upload

You can upload files to the server by sending a POST request to /upload. The uploaded files will be saved in the specified directory. This feature allows users to easily upload and manage files on the server.

File Download

You can download files that have been uploaded to the server from the specified download directory. Access the files via the /downloads endpoint. This feature provides a simple way to retrieve files stored on the server.

Configuration

The server reads configuration from a config.json file. You can customize various aspects of the server's behavior, including security settings, performance optimization, and more.
